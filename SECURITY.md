# Security Policy

## Supported Versions

Security fixes are applied to the latest released version on `main`.

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected vulnerabilities.

Use GitHub's private vulnerability reporting flow for this repository when available. If private reporting is unavailable, contact the maintainer privately through the repository owner's GitHub profile before sharing details publicly.

When reporting a vulnerability, please include:

- A short description of the issue and affected area
- Reproduction steps or a minimal proof of concept
- Impact assessment if known
- Any suggested mitigation or patch if you have one

We aim to acknowledge new reports within 5 business days and will coordinate disclosure after a fix is available.
